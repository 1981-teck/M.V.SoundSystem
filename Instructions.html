SKorganizer Pro - Detailed User Guide

Welcome to the SKorganizer Pro user guide! This document will walk you through all of the application's features, helping you set up and make the most of its file organization capabilities. It's recommended to launch SkOrganizer Pro in administrator mode to avoid file permission issues.

Required Specifications:

Operating System: Windows 10, macOS (Darwin), Linux.

Hardware:

Minimum: A modern dual-core processor (e.g., Intel Core i3 or AMD Ryzen 3).

Recommended: A quad-core processor or higher (Intel Core i5/i7 or AMD Ryzen 5/7) for optimal performance, especially when organizing a large number of files.

Memory (RAM):

Minimum: 4 GB RAM.

Recommended: 8 GB RAM or more to ensure smooth operation, especially if you are using other applications simultaneously or managing large amounts of data.

Storage Space: Hard Drive (HDD/SSD): Little space required for the program installation itself (a few MB). However, you need sufficient hard drive space for the files the program will organize. Using a Solid State Drive (SSD) is strongly recommended to improve overall file operation speed.

Display: Minimum resolution: 1024x768 pixels.

1. Getting Started

Upon first launch, you'll see the application's main interface, divided into several tabs for easy navigation and configuration.

Navigating the Interface

The user interface is based on main Menus, which will include:

  • File-Manage Folders: To add, configure, and manage the drives or folders to be organized.
  • File-Global Settings: To define how your files should be classified and moved, and to change the application's theme and behavior.
  • Edit: To view past operations and undo them.
  • Help: To view a summary of features.
Warning: Do not select the main drive (e.g., C:), as this would compromise the operating system's functionality if the "RECURSIVE" sorting system is used. IMPORTANT: Check that the files to be organized do not have the "block" for files coming from another computer. If you are sure of their origin: right-click on the file, properties - uncheck the block. SkOrganizer Pro cannot process blocked files; this is to protect your computer.

2. Folder Configuration

This is the first step to start organizing. You need to tell SKorganizer Pro which folders you want to manage.

Adding and Removing Folders

  1. In the Manage Folders tab, use the "Add" button.
  2. Select the source folder you wish to organize or monitor.
  3. For each added folder, you can specify if it will be designated for **Downloads (specific behavior) or General**, where files will be organized.
  4. Use the "Settings" button to choose various settings for that folder, for example: Folder Type - Organization Type (extension, date, alphabetical, etc).
  5. Use the "Remove Folder" button to delete an existing configuration.
Tip: Start with a "Downloads" folder or a test folder to familiarize yourself with the process before applying it to more important directories. Backups are recommended for large recursive operations on new folders or drives with existing files (and subfolders).

3. Defining Organization Rules

Rules determine how SKorganizer Pro will classify and move your files.

Available Rule Types:

  • Organization by Extension (Default):

    This is the simplest mode. SKorganizer Pro has a predefined list of categories (e.g., Images, Documents, Videos) and associated file extensions. Files will be moved to the appropriate destination folder based on their extension.

    Example: A .jpg file will go into the "Images" folder.

  • Alphabetical Organization:

    Organizes files based on their initial letter by creating folders for each different initial present.

    Example: Files Anna.txt Alberto.jpg will be moved to a folder "A".

  • Organization by Date:

    Organizes files based on their oldest date (creation/modification). You can choose an automatic date format (YYYY-MM-DD) or define a custom one.

    Example: Files created/modified on March 15, 2025, will go into a folder "2025-03-15".

  • Organization by Size:

    Define size ranges (e.g., Less than 1MB, Between 1MB and 10MB, Greater than 100MB) and assign a destination folder for each range.

    Example: Large files (e.g., > 1GB) can be moved to an "Large Archives" folder.

  • Organization by Name Pattern:

    Allows you to define regular expressions (regex) to match file names. Files that match the pattern will be moved to a specified folder.

    Example: If you create a pattern for report_.*\.pdf, all PDFs with names starting with "report_" will go into an "Annual Reports" folder.

    Tip: Regular expressions are powerful. If you are unfamiliar, search for online guides or start with simple patterns like .*project_name.*.
  • Custom Rules:

    Manually map specific file extensions to custom destination folders, regardless of predefined categories.

    Example: You can decide that all .log files go into the "System Logs" folder.

  • "Other Files" Folder (extension) "#" (alphabetical):

    Files that do not match any of the defined rules will be moved to a designated folder for "Other Files" or "#" (configurable in settings).

4. Executing Organization

Once you've configured your folders and rules, you're ready to start the organization process.

Manual Start

In the main window, select a folder with a right-click, then select "organize now". If you want to organize all folders in the list, select "organize all now". By clicking this, SKorganizer Pro will start scanning and moving files in the main folder (for subfolders, select reorganize recursively) according to your rules.

Automatic Monitoring

If you have enabled monitoring for a folder (right-click) or for all folders in the main window (Start All Monitoring), organization will occur automatically in the background whenever a new file is added or modified in that folder.

Status and Notifications

The application will provide updates on the organization status in the log window, showing moved files and completed folders. For long operations, a progress bar will appear.

5. Duplicate File Management

SKorganizer Pro can detect duplicate files (same name) and identical files (same content) using hashing algorithms to ensure accuracy.

Conflict Resolution

When a duplicate is detected, the application will act according to your Conflict Resolution Policy, configurable in the global settings:

  • Rename: (default) The duplicate file will be renamed (e.g., document_copy_1.pdf) and moved.
  • Overwrite: The existing file at the destination will be replaced with the new file. (Use with caution!)
  • Ask: The application will ask you how to proceed for each duplicate detected. (Not recommended for large file collections)
  • Skip: The duplicate file will not be moved and will remain in the source folder.
Warning: The application uses cryptographic hashing algorithms (like MD5, SHA1, SHA256, BLAKE2B) to generate a unique digital fingerprint for the content of each file. If two files have the same hash, SKorganizer Pro considers them IDENTICAL in their content, regardless of name or location, and moves them to the recycle bin. If they are merely DUPLICATES, it will act according to the selected option (it is recommended to keep "default").

6. Using History and Undo Function

SKorganizer Pro tracks all move operations, offering you the ability to undo them.

Viewing History

In the Application Log tab, you can see a list of all organization operations performed, including moved files and their original locations.

Undoing Operations

In the Edit Menu, you can select: Undo Last Move or Undo Operations to restore files to their original positions. The application uses the "System Recycle Bin" for file deletion.

Recovery Folder

The program has a recovery folder where it saves files in case of errors or problems with moving and undoing operations. This is located in the system's main drive, users/"username"/Recovery.

Tip: Set a reasonable limit for the history size in global settings to avoid excessive memory consumption (only modify this setting if you are an expert, otherwise leave the "default" setting). Warning: If the system recycle bin has the "Don't move files to the Recycle Bin. Remove files immediately" option active, this will also affect the program. It is recommended to disable it.

7. Global Settings

The File Global Settings menu allows you to customize the general behavior of SKorganizer Pro.

  • GENERAL:
  • Logging Level: Choose the level of detail for application logs (useful for troubleshooting).
  • Theme: Select your preferred application color.
  • Max Worker Threads: Defines the maximum number of operations that can be performed simultaneously.
  • Max file size for hash: Defines the maximum size of a file (in MB or GB) for which the application will calculate a hash value to determine if files are identical.
  • Hash timeout: Defines the maximum time (in seconds) the application will wait to complete the hash calculation of a single file.
  • Exclusion Suffix: Indicates the suffix files must have to be ignored by monitoring.
  • 'Other Files' Folder: Indicates the folder where unrecognized files will be moved.
  • Temporary Extensions: Allows you to tell SKorganizer Pro which file types to consider "temporary" or unimportant for main organization, enabling it to manage them appropriately (usually by ignoring or deleting them) to maintain order.
  • Use Recycle Bin: Decide whether files moved or deleted by SKorganizer Pro should go to the system recycle bin or be permanently removed.
  • History Size: Set the maximum number of operations to keep in history and/or the maximum size in MB.
  • Custom Date Format: If you've chosen organization by date, you can define a specific date format here.
  • CLEANUP:
  • Max File Size for Cleanup: Indicates the maximum size of files considered by the Automatic Cleanup function (right-click on the folder in the main window).
  • Recycle Bin Path: Select a custom path.
  • Use recycle bin for deletions: Deselecting is not recommended; if deselected, it will not use the recycle bin.
  • BEHAVIOR:
  • Max Undoable Operations: Customize how many undoable operations are stored (default 500, calculated for an optimal security-size ratio).
  • Max history size: Customize how much maximum space the history can use.
  • Conflict Resolution: Select what the application should do when it finds duplicate files.
  • Deduplicate files: Selecting "Deduplicate files" enables an important control mechanism that helps you maintain data integrity and optimize disk space, while deselecting it disables this control, allowing files to be moved without content duplicate checks.
  • Hash Algorithm: Select a custom algorithm to verify duplicate files.

8. Troubleshooting and Useful Tips

  • Excluded Files: Files with a specific suffix (e.g., .skorganizer_ignore) can be automatically excluded from organization. Check settings for this option.
  • Error Policies: SKorganizer Pro has internal policies for handling file move errors (e.g., retry or move to a quarantine folder). This helps prevent data loss.
  • SkOrganizer was designed to manage files, errors, and all processes with sophisticated systems, preventing errors, using the system recycle bin for simplicity, and employing effective recovery and undo systems. Therefore, it is recommended not to modify the program's options unless you fully understand the mechanics of such modifications, or at least to make such modifications and test them on folders created specifically to avoid potential blockages or data loss.
  • Final recommendations: Please read the Features Details file and EULA.

9. RECURSIVE Function

  • The Recursive function is used to perform organization on existing folders, including Subfolders, or if you decide to change the organization method. In this case, a confirmation window will appear for the user. Please make any necessary backups of important files/folders before performing this action. The program may need "time" to execute operations on "very large" quantities of files or on less powerful computers. Incorrect interactions, blackouts, etc., could cause data loss WITHOUT possibility of recovery. ALL files deleted by this function will be in the "system recycle bin" to offer an additional security system.
